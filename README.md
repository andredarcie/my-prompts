# My Prompts

Prompts diretos para desenvolvimento de software, arquitetura e boas práticas.

## Categorias

| Pasta | Conteúdo |
|-------|----------|
| [`dev/`](dev/) | Code review, refatoração, debugging, testes |
| [`architecture/`](architecture/) | Design de sistemas, decisões técnicas, trade-offs |
| [`best-practices/`](best-practices/) | Patterns, clean code, segurança, performance |

## Como usar

1. Navegue até a categoria desejada
2. Abra o prompt no GitHub Pages
3. Clique em **Copiar** e cole no seu LLM preferido

## Convenções

- Prompts em blocos de código para facilitar a cópia
- Nomes de arquivo descritivos em `kebab-case`
- Um prompt por arquivo, com contexto mínimo necessário
- Foco em output acionável, sem fluff
